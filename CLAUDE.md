> - `lib/src/flutter_lucide.dart` (~2 MB, 1700+ icons) — generated, never hand-edited

---

## What this package is

A Flutter package that exposes every [Lucide](https://lucide.dev/) icon as a tree-shakeable `static const IconData` constant inside `abstract final class LucideIcons`.

Users write: `Icon(LucideIcons.heart)` — nothing more needed.

```

  flutter_lucide.dart          # one-liner barrel: exports src/flutter_lucide.dart

pubspec.yaml                   # version, description, font asset declaration
CHANGELOG.md                   # release history
README.md                      # pub.dev landing page
example/lib/main.dart          # showcase app
```

## How the generation script works

Run from `script/` directory:
```bash
dart lib/script.dart
mv flutter_lucide_update.txt ../lib/src/flutter_lucide.dart
```

### What it does, step by step

1. **Reads `script/info.json`** — unicode map for every icon:
   ```json
   { "heart": { "encodedCode": "\\ea4c", "prefix": "icon", "className": "icon-heart" } }
   ```
   `encodedCode` `\\eXXX` → Dart codepoint `0xeXXX` (script replaces `\\` with `0x`).

2. **Reads every `.json` in `script/icons/`** (sorted alphabetically) — per-icon metadata:
   ```json
   {
     "contributors": ["colebemis", "ericfennis"],
     "tags": ["love", "like", "emotion"],
     "categories": ["emoji", "social"]
   }
   ```

3. **Reads matching `.svg` from `script/icons/`** — applies these transforms then base64-encodes for the dartdoc `![preview]()` image:
   - stroke color → `#eb1d25`
   - width/height → `48`
   - adds `xmlns` and `version="1.1"` if missing

4. **Name conversion**: `kebab-case` filename → `snake_case` Dart identifier
   - `arrow-big-down` → `arrow_big_down`
   - `zap-off` → `zap_off`

5. **Generates one constant per icon**:
   ```dart
   /// Represents the [heart] icon from the Lucide icon set.
   ///
   /// ![heart](data:image/svg+xml;base64,<base64>)
   ///
   /// Description:
   /// - The [heart] icon is a graphical symbol ... related to love, like, emotion.
   /// - It belongs to the categories: emoji, social
   ///
   /// Acknowledgements:
   /// - Contributors: [https://github.com/colebemis](...)
   static const IconData heart = IconData(0xea4c, fontFamily: _fontFamily, fontPackage: _fontPackage);
   ```

6. **Wraps everything** in:
   ```dart
   // ignore_for_file: constant_identifier_names
   import 'package:flutter/widgets.dart';

   @staticIconProvider
   abstract final class LucideIcons {
     const LucideIcons._();
     static const String _fontFamily = 'lucide';
     static const String _fontPackage = 'flutter_lucide';
     // ... all icon constants
   }
   ```

7. **Writes** output to `script/flutter_lucide_update.txt`, then you `mv` it.

### Why `@staticIconProvider`?
Tells Flutter's tree shaker that this class only contains static `IconData` constants, enabling dead-code elimination of unused icons.

### Why no `fromCode()` constructor?
It was removed — a dynamic factory prevents tree shaking. See [issue #3](https://github.com/ravikovind/flutter_lucide/issues/3).

## How to look up a specific icon (without reading the dart file)

**First stop: [`ICONS.md`](ICONS.md)** — a flat, searchable table of all icons with Dart constant name, unicode, tags, categories, and contributors. Grep it or read it directly.

```bash
grep "heart" ICONS.md       # find by icon name
grep "astrology" ICONS.md   # find by tag
grep "social" ICONS.md      # find by category
```

For programmatic lookups:

1. **Check if name exists**:
   ```bash
   python3 -c "import json; d=json.load(open('script/info.json')); print('heart' in d)"
   ```

2. **Get its codepoint**:
   ```bash
   python3 -c "import json; d=json.load(open('script/info.json')); print(d['heart']['encodedCode'])"
   # \ea4c  →  Dart: 0xea4c
   ```

3. **Get its metadata**:
   ```bash
   cat script/icons/heart.json
   ```

4. **Count total icons**:
   ```bash
   python3 -c "import json; print(len(json.load(open('script/info.json'))))"
   ```

`ICONS.md` is auto-generated by the release script — always up to date with the latest Lucide version.

## Versioning convention

**From v1.11.0 onwards, the Flutter package version mirrors the Lucide version exactly.**

| Lucide version | Flutter package version |
|----------------|------------------------|
| 0.562.0        | 1.8.x (legacy)         |
| 1.8.0          | 1.9.0 (legacy)         |
| 1.11.0         | **1.11.0** ← synced from here |
| 1.12.0         | 1.12.0                 |
| 2.0.0          | 2.0.0                  |

Rule: **Flutter package version = Lucide version**. Always match exactly.

## How to release a new Lucide version

### 1. Download from [Lucide releases](https://github.com/lucide-icons/lucide/releases)
- `lucide-font-<version>.zip` → extract, replace:
  - `script/info.json`
  - `lib/fonts/lucide.ttf`
- `lucide-icons-<version>.zip` → extract, replace entire `script/icons/` directory

### 2. Regenerate the Dart file and ICONS.md
```bash
cd script/
dart lib/script.dart
mv flutter_lucide_update.txt ../lib/src/flutter_lucide.dart
```
This also auto-generates `ICONS.md` in the repo root — no extra step needed.

### 3. Find new / removed / renamed icons
```bash
diff \
  <(git show HEAD:lib/src/flutter_lucide.dart | grep "static const IconData" | sed "s/.*static const IconData //; s/ =.*//") \
  <(grep "static const IconData" lib/src/flutter_lucide.dart | sed "s/.*static const IconData //; s/ =.*//")
# Lines with > are new, lines with < are removed
```

Count icons:
```bash
grep -c "static const IconData" lib/src/flutter_lucide.dart
```

### 4. Update these four files

| File | What to change |
|------|---------------|
| `pubspec.yaml` | `version` and `description` icon count |
| `CHANGELOG.md` | New entry at top (see format below) |
| `README.md` | Icon count, Lucide version, install version |
| `script/README.md` | Icon count in Output section |

### 5. Commit, push, publish
```bash
git add -A
git commit -m "feat: bump to Lucide <version> (v<version>)"
git push origin main
dart pub publish --dry-run
dart pub publish
```
